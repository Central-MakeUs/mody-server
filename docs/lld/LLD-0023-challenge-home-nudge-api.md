## 1. 목적 / 배경

챌린지 홈은 그룹 단위 활동 요약을 보여주고, 오늘 기록하지 않은 버디에게 기록을 유도하는 진입점이다.
클라이언트는 홈 진입 시 그룹 참여 상태를 검증받고, 월간 활동 수치와 버디별 오늘 기록 여부를 조회해야 한다.

## 2. 범위

### In scope

- 챌린지 홈 요약 조회.
- 버디 찌르기 대상 조회.
- 버디 찌르기 알림 요청.
- 그룹 참여 권한 검증.
- Swagger 성공/예외 응답 문서화.

### Out of scope

- 버디 찌르기 일일 횟수 제한 정책.
- 찌르기 발송 이력 별도 조회.
- 챌린지 홈 캐싱 또는 통계 배치.
- 주간 챌린지 공유 이미지 생성.

## 3. API 동작

### 챌린지 홈 요약 조회

```http
GET /api/v1/groups/{groupId}/challenges/summary
```

1. access token의 회원이 활성 회원인지 확인한다.
2. `groupId`가 활성 그룹인지 확인한다.
3. 요청 회원이 그룹에 참여 중인지 확인하고, 회원의 `joinedAt`을 기준으로 함께한 일수를 계산한다.
4. 이번 달의 그룹 기록을 조회한다.
5. 현재 참여 중인 모든 그룹원이 같은 날짜에 1개 이상 기록한 날짜 수를 `allMemberRecordedDays`로 반환한다.
6. 이번 달 운동 기록의 `exerciseDurationMinutes` 합계를 반환한다.
7. 이번 달 `COMPLETED` 상태가 된 그룹 챌린지 개수를 반환한다.

### 버디 찌르기 대상 조회

```http
GET /api/v1/groups/{groupId}/challenges/nudges
```

1. 요청 회원의 그룹 참여 여부를 검증한다.
2. 본인을 제외한 현재 참여 중인 그룹원을 가입일 순서로 조회한다.
3. 오늘 기록이 1개 이상 있으면 `recordedToday = true`로 반환한다.
4. 프로필 이미지는 저장소 base URL과 imageKey를 조합해 반환한다.

### 버디 찌르기

```http
POST /api/v1/groups/{groupId}/challenges/nudges/{memberId}
```

1. 요청 회원과 대상 회원이 모두 같은 그룹에 참여 중인지 확인한다.
2. 본인을 찌르는 요청은 `CHALLENGE301`로 거절한다.
3. `NotificationRequestService`에 `BUDDY_NUDGE` 알림 요청을 발행한다.
4. 알림 중복 제어는 알림 아웃박스의 dedupe key 정책을 따른다.

## 4. 예외 코드

- `AUTH401`~`AUTH405`: 인증 실패.
- `MEMBER302`: 회원 없음.
- `GROUP302`: 그룹 없음.
- `GROUP306`: 그룹 참여 정보 없음.
- `CHALLENGE301`: 본인 찌르기 등 챌린지 요청값 검증 실패.

## 5. 테스트 시나리오

- 챌린지 홈 요약은 회원의 그룹 가입일, 월간 운동 시간, 전원 기록 일수, 완료 챌린지 수를 반환한다.
- 버디 찌르기 대상은 본인을 제외하고 오늘 기록 여부와 프로필 URL을 반환한다.
- 버디 찌르기는 대상 회원에게 알림 요청을 발행한다.
- 본인을 찌르면 `CHALLENGE301`을 반환한다.
- Swagger에 성공/예외 응답이 생성된다.

## 6. 미결정 사항 (Open Questions)

- 버디 찌르기 일일 횟수 제한은 정책 확정 후 알림 dedupe 또는 별도 이력 테이블로 확장한다.
- 그룹 홈 통계를 현재 JOINED 멤버 기준으로 계산한다. 탈퇴 회원 포함 정책이 필요하면 별도 통계 기준을 추가한다.
