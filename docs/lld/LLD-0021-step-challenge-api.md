## 1. 목적 / 배경

걸음수 챌린지는 그룹 단위로 하나의 진행 중인 챌린지를 가진다.
앱은 현재 진행률, 걸어간 지역 내역, 회원별 기여도 순위, 챌린지 변경을 조회/실행할 수 있어야 한다.

## 2. 범위

### In scope

- 현재 걸음수 챌린지 현황 조회.
- 완료/변경 종료된 걸어간 지역 내역 조회.
- 변경 가능한 걸음수 챌린지 목록 조회.
- 현재 걸음수 챌린지 기여도 순위 조회.
- 걸음수 챌린지 변경 및 진행률 초기화.
- 그룹 구성원 권한 검증.
- Swagger 성공/예외 응답 문서화.

### Out of scope

- 걸음수 데이터 동기화 API.
- 걸음수 챌린지 완료 자동 판정 배치.
- 완료 알림 발송 연결.

## 3. 데이터 모델

- `challenge`: 챌린지 마스터. `challengeType = STEP`.
- `step_challenge_detail`: 출발지, 도착지, 거리, 목표 걸음수, 노출 순서.
- `group_challenge`: 그룹에서 진행 중이거나 완료/변경 종료된 챌린지 인스턴스.
- `step_record`: 그룹 챌린지, 회원, 날짜별 걸음수 기록.

걸음수 챌린지 마스터 데이터는 DB에서 관리한다. 현재 프로젝트는 Flyway를 쓰지 않으므로,
애플리케이션 시작 시 `StepChallengeMasterDataInitializer`가 아래 6개 마스터 데이터를 보장한다.
운영에서 seed를 끄고 싶으면 `CHALLENGE_STEP_MASTER_DATA_INITIALIZER_ENABLED=false`를 주입한다.

## 4. API 동작

### 현재 걸음수 챌린지 조회

1. 요청 회원이 그룹에 참여 중인지 확인한다.
2. 그룹의 `IN_PROGRESS` 상태 걸음수 `group_challenge`를 조회한다.
3. 해당 group challenge의 `step_record.stepCount`를 합산한다.
4. 챌린지명, 목표 걸음수, 현재 걸음수를 반환한다.

현재 진행 중인 걸음수 챌린지가 없으면 `CHALLENGE303`을 반환한다.

### 지금까지 걸어간 지역 조회

1. 요청 회원이 그룹에 참여 중인지 확인한다.
2. `COMPLETED` 상태의 걸음수 group challenge를 조회한다.
3. 각 챌린지의 도착지를 지역명으로 반환한다.
4. 지역 이미지는 현재 스토리지 키 정책이 없어 `regions/{destination}.png` 규칙으로 생성한다.

### 변경 가능한 걸음수 챌린지 목록 조회

1. 요청 회원이 그룹에 참여 중인지 확인한다.
2. `STEP` 타입 챌린지와 `step_challenge_detail`을 조회한다.
3. `step_challenge_detail.displayOrder` 오름차순으로 반환한다.
4. 현재 그룹에서 진행 중인 챌린지는 `selected = true`로 표시한다.

### 기여도 순위 조회

1. 현재 진행 중인 걸음수 group challenge를 조회한다.
2. 활성 그룹 멤버 목록을 조회한다.
3. `step_record`를 회원별 합산한다.
4. 걸음수 내림차순, 가입 순서 오름차순, 회원 id 오름차순으로 정렬한다.
5. 동점자는 같은 순위를 쓰지 않고 정렬 순서 기준 1, 2, 3 순위를 부여한다.

### 챌린지 변경

1. 요청 회원이 그룹에 참여 중인지 확인한다.
2. 변경 대상 `challengeId`가 STEP 타입인지 확인한다.
3. 현재 진행 중인 챌린지와 같은 `challengeId`면 기록을 초기화하지 않고 현재 상태를 반환한다.
4. 다른 챌린지로 변경하면 기존 진행 중인 걸음수 group challenge를 `RESET` 처리하고 `endedAt`을 기록한다.
5. 새 `group_challenge`를 `IN_PROGRESS`로 생성한다.
6. 새 챌린지 진행률은 0부터 시작한다.

## 5. 예외 코드

- `AUTH401`~`AUTH405`: 인증 실패.
- `MEMBER302`: 회원 없음.
- `GROUP302`: 그룹 없음.
- `GROUP306`: 그룹 참여 정보 없음.
- `CHALLENGE301`: 챌린지 요청값 검증 실패.
- `CHALLENGE302`: 챌린지를 찾을 수 없음.
- `CHALLENGE303`: 진행 중인 걸음수 챌린지 없음.

## 6. 테스트 시나리오

- 현재 걸음수 챌린지의 목표/현재 걸음수를 계산한다.
- 완료/변경 종료된 지역 내역을 조회한다.
- 변경 가능한 걸음수 챌린지 목록을 노출 순서와 현재 선택 여부로 조회한다.
- 현재 챌린지 기여도 순위를 회원별 걸음수 합산으로 계산한다.
- 챌린지 변경 시 기존 진행 챌린지는 `RESET`, 새 챌린지는 `IN_PROGRESS`가 된다.
- 같은 챌린지로 변경 요청하면 기존 진행률을 유지한다.
- 그룹 미참여 회원은 챌린지 API에 접근할 수 없다.
- 걸음수 챌린지 마스터 데이터 초기화는 중복 생성하지 않는다.
- Swagger에 성공/예외 응답이 생성된다.

## 7. 미결정 사항 (Open Questions)

- 지역 이미지 URL을 DB 컬럼으로 둘지, 현재처럼 규칙 기반 key로 유지할지 결정 필요.
- 걸음수 챌린지 완료 자동 판정 시점과 알림 발송 연결 방식 결정 필요.
