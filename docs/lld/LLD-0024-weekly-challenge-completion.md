# LLD-0024: 주간 챌린지 완료 판정 및 알림

> Low-Level Design. 이 문서는 #75 구현과 PR 본문의 오라클이다.

| 항목 | 값 |
| --- | --- |
| 상태 | Accepted |
| Issue | #75 |
| 관련 ADR | ADR-0012, ADR-0018 |
| 작성자 | Codex |
| 작성일 | 2026-07-06 |

## 1. 목적 / 배경

주간 챌린지 인증은 사진을 저장하지만, 전원이 인증했을 때 그룹 챌린지를 완료 상태로 전환하지 않는다.
완료 상태가 없으면 챌린지 홈의 월간 완료 챌린지 수와 주간 챌린지 완료 알림이 실제 인증 흐름과 연결되지 않는다.

## 2. 범위

### In scope

- 주간 챌린지 인증 저장 후 완료 조건 판정.
- 현재 참여 중인 그룹원 전원이 인증하면 `group_challenge`를 `COMPLETED`로 변경.
- 완료 시 `WEEKLY_CHALLENGE_COMPLETED` 알림 요청 발행.
- 이미 완료된 주간 챌린지에 대한 추가 인증 거절.
- 서비스 테스트 및 Swagger 예외 응답 문서화.

### Out of scope

- 주간 챌린지 자동 배정/생성 배치.
- 공유용 이미지 합성.
- 탈퇴 회원을 포함한 과거 기준 완료 재계산.

## 3. 완료 판정 규칙

1. 인증 요청 회원이 그룹에 현재 참여 중인지 확인한다.
2. 대상 `groupChallengeId`가 PHOTO 타입 주간 챌린지인지 확인한다.
3. 대상 그룹 챌린지가 이미 `COMPLETED`라면 `CHALLENGE305`를 반환한다.
4. 회원이 이미 같은 그룹 챌린지에 인증했다면 `CHALLENGE304`를 반환한다.
5. 인증 이미지를 저장한다.
6. 현재 JOINED 그룹원 수와 해당 그룹 챌린지 인증 수를 비교한다.
7. 인증 수가 현재 JOINED 그룹원 수 이상이면 그룹 챌린지를 완료 처리한다.
8. 완료 처리 직후 `NotificationRequestService`로 `WEEKLY_CHALLENGE_COMPLETED` 알림 요청을 발행한다.

## 4. 알림 연결

- 알림 타입: `WEEKLY_CHALLENGE_COMPLETED`
- 수신자: 알림 수신자 resolver 기준의 현재 JOINED 그룹원 전체
- 중복 제어: 알림 아웃박스 dedupe key가 `type + groupChallengeId + receiverMemberId`로 중복 저장을 방지한다.

## 5. 예외 코드

- `AUTH401`~`AUTH405`: 인증 실패.
- `MEMBER302`: 회원 없음.
- `GROUP302`: 그룹 없음.
- `GROUP306`: 그룹 참여 정보 없음.
- `CHALLENGE302`: 챌린지를 찾을 수 없음.
- `CHALLENGE304`: 이미 주간 챌린지 인증을 완료함.
- `CHALLENGE305`: 이미 완료된 챌린지임.

## 6. 테스트 시나리오

- 인증 후 아직 전원이 인증하지 않았으면 완료 처리하지 않는다.
- 마지막 그룹원이 인증하면 그룹 챌린지를 완료 처리하고 완료 알림을 요청한다.
- 이미 완료된 주간 챌린지는 추가 인증할 수 없다.
- Swagger에 완료된 챌린지 예외 응답이 생성된다.

## 7. 미결정 사항 (Open Questions)

- 완료 판정은 현재 JOINED 그룹원 기준이다. 탈퇴 회원 포함 정책이 필요하면 별도 통계 재계산 정책을 추가한다.
- 동시에 여러 명이 마지막 인증을 시도하는 레이스는 DB 제약 또는 잠금 전략을 별도 이슈에서 보강한다.
