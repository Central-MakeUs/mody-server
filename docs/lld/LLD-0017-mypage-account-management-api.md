# LLD-0017: 마이페이지 회원 관리 API

> Low-Level Design. 이 문서는 마이페이지 회원 관리 API 구현과 PR 본문의 **오라클(ground truth)** 이다.

| 항목 | 값 |
| --- | --- |
| 상태 | Accepted |
| Issue | #46 |
| 관련 ADR | ADR-0003, ADR-0009 |
| 작성자 | Codex |
| 작성일 | 2026-07-04 |

## 1. 목적 / 배경

마이페이지에서 회원 탈퇴와 그룹 나가기가 실제 회원 상태와 조회 결과에 반영되어야 한다.
탈퇴하거나 그룹에서 나간 뒤에는 해당 회원의 정보, 기록, 댓글, 알림이 앱 조회 API에 노출되지 않아야 한다.

## 2. 범위

### In scope

- 회원 탈퇴 API 실제 구현.
- 탈퇴 시 refresh token, 소셜 계정, 체중, 알림 설정, 운동 일정, 알림, 푸시 토큰, 그룹 멤버십, 기록, 댓글, 기록 조회 이력, 회원별 챌린지 인증/걸음수 기록 삭제 처리.
- 그룹 나가기 API 실제 구현.
- 그룹 나가기 시 해당 그룹 안에서 작성한 기록, 댓글, 기록 조회 이력, 회원별 챌린지 인증/걸음수 기록 삭제 처리.
- 그룹 구성원 조회 API를 그룹 서비스 기반 실제 구현으로 연결.
- Swagger 성공/주요 예외 응답 명세.

### Out of scope

- 회원 탈퇴 복구.
- 스토리지 이미지 객체 삭제.
- 탈퇴 사유 저장.
- 실제 푸시 발송 취소.
- 물리 삭제 배치.

## 3. 인터페이스 / API

모든 API는 `Authorization: Bearer {accessToken}`을 사용한다.

```http
DELETE /api/v1/mypage/me
GET /api/v1/mypage/groups/{groupId}/members
DELETE /api/v1/mypage/groups/{groupId}/members/me
```

로그아웃은 마이페이지 전용 API를 새로 만들지 않고 기존 Auth API를 사용한다.

```http
POST /api/v1/auth/logout
```

## 4. 데이터 모델

다음 엔티티는 `BaseEntity.delete()`를 사용해 soft delete한다.

- `member`
- `social_account`
- `refresh_token`
- `weight_record`
- `notification_setting`
- `exercise_schedule`
- `notification`
- `member_push_token`
- `group_member`
- `activity_record`
- `record_comment`
- `record_view_history`
- `challenge_proof`
- `step_record`

다음 데이터는 회원별 생성/기여 데이터가 아니므로 삭제하지 않는다.

- `challenge`
- `step_challenge_detail`
- `group_challenge`
- `challenge_badge`

외래키 제약조건은 사용하지 않고 id 값으로만 참조한다.

## 5. 처리 흐름

### 회원 탈퇴

1. `@CurrentMember`가 access token에서 회원 id를 추출한다.
2. 활성 회원인지 검증한다.
3. 회원의 활성 refresh token을 모두 삭제한다.
4. 회원의 소셜 계정, 체중 기록, 알림 설정, 운동 일정, 알림, 푸시 토큰을 삭제한다.
5. 회원이 참여 중인 모든 그룹 멤버십을 나감 처리한다.
6. 회원이 작성한 모든 기록을 삭제한다.
7. 회원이 작성한 모든 댓글을 삭제한다.
8. 회원이 작성한 기록에 달린 댓글도 함께 삭제한다.
9. 회원이 조회자 또는 작성자로 포함된 기록 조회 이력을 삭제한다.
10. 회원이 업로드한 주간 챌린지 인증과 걸음수 기록을 삭제한다.
11. 회원 엔티티를 삭제한다.

### 그룹 나가기

1. 회원이 해당 그룹에 참여 중인지 검증한다.
2. 해당 그룹 멤버십을 나감 처리한다.
3. 해당 그룹에서 회원이 작성한 기록을 삭제한다.
4. 해당 그룹에서 회원이 작성한 댓글을 삭제한다.
5. 회원이 해당 그룹에서 작성한 기록에 달린 댓글도 함께 삭제한다.
6. 해당 그룹에서 회원이 조회자 또는 작성자로 포함된 기록 조회 이력을 삭제한다.
7. 해당 그룹의 그룹 챌린지에 연결된 회원의 주간 챌린지 인증과 걸음수 기록을 삭제한다.

## 6. 조회 노출 정책

- 기존 피드/상세 조회는 활성 그룹 멤버와 `deletedAt is null` 조건을 기준으로 한다.
- 탈퇴 또는 그룹 나가기 이후에는 삭제 처리된 기록/댓글/멤버십/조회이력/챌린지 기여 데이터가 조회되지 않는다.
- 그룹 구성원 조회는 `GroupService.getGroupMembers`를 사용해 현재 참여 중인 회원만 반환한다.

## 7. 예외 / 에러 처리

- 인증 실패: `AUTH401`~`AUTH405`.
- 회원 없음: `MEMBER302`.
- 그룹 없음: `GROUP302`.
- 그룹 참여 정보 없음: `GROUP306`.

## 8. 인수조건 (Acceptance Criteria)

- [x] 인증 회원이 탈퇴할 수 있다.
- [x] 탈퇴 시 회원의 refresh token과 소셜 계정이 삭제 처리된다.
- [x] 탈퇴 시 회원 관련 기록/댓글/알림/설정/푸시 토큰/그룹 멤버십/조회이력/챌린지 기여 데이터가 삭제 처리된다.
- [x] 인증 회원이 참여 중인 그룹을 나갈 수 있다.
- [x] 그룹 나가기 시 해당 그룹 안에서 작성한 기록/댓글/조회이력/챌린지 기여 데이터가 삭제 처리된다.
- [x] 그룹 구성원 조회가 실제 그룹 서비스 결과를 반환한다.
- [x] Swagger에 성공/주요 예외 응답이 반영된다.
- [x] 관련 테스트가 통과한다.

## 9. 영향 범위 / 마이그레이션

- 마이페이지 회원 관리 stub API가 DB 기반 구현으로 교체된다.
- 삭제 처리는 soft delete이므로 기존 테이블 구조를 변경하지 않는다.
- 이미지 객체 삭제는 이번 범위에서 수행하지 않으며, 추후 스토리지 정리 배치에서 처리한다.

## 10. 미결정 사항 (Open Questions)

- 법적 보관이 필요한 데이터가 생기면 탈퇴 삭제 범위를 재검토한다.
